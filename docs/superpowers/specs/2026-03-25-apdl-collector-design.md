# APDL Collector Design

## Overview

Design a transparent wrapper around existing APDL `Commands` classes that collects generated command strings instead of executing them against an ANSYS instance. The collector accumulates commands in memory and writes them to a file on demand.

## Architecture

### Core Class: APDLCollector

```
APDLCollector
    ├── commands: Commands       # Existing command object
    ├── command_log: List[str]  # Accumulated command strings
    ├── processor_stack: List[str]  # Tracks active processors for context
    │
    ├── run(cmd: str) -> None   # Collects command instead of executing
    ├── to_string() -> str      # Returns all commands as single string
    ├── to_file(path: str) -> None  # Writes commands to file
    │
    └── Context managers: prep7(), post1(), post26(), solution(), finish()
```

### Design Principles

- **Transparent wrapping**: `APDLCollector` holds a `Commands` instance and delegates method calls. The `Commands` class itself is not modified.
- **Non-invasive**: Existing `Commands` implementation is unchanged. Only the execution backend (`run()`) is intercepted.
- **Standard context managers**: Use Python's `contextlib` for processor entry/exit blocks.

## Command Collection Flow

1. User creates `collector = APDLCollector()`
2. Commands are called via `collector.commands.k(1, 0, 0, 0)` (or delegated shortcuts)
3. Each command's `run()` method appends the command string to `command_log`
4. On `to_file()` call, all collected commands are written to disk

### How run() is Redirected

The existing `Commands.run()` method executes APDL commands via gRPC/gUI connection. The collector wraps `Commands` and provides its own `run()` that:

```python
def run(self, cmd: str, **kwargs):
    self.command_log.append(cmd)
```

This replaces the execution call with simple accumulation.

## Processor Context Management

APDL requires explicit processor entry/exit commands:
- `/PREP7` — enter preprocessor
- `/POST1` — enter postprocessor
- `/POST26` — enter time-history postprocessor
- `/SOLU` — enter solution processor
- `FINISH` — exit current processor

Context managers automate these:

```python
@contextmanager
def prep7(self):
    self.run("/PREP7,")
    self.processor_stack.append("prep7")
    yield
    self.processor_stack.pop()
    self.run("FINISH,")
```

## File Output Format

Generated files include a header:

```apdl
! Generated by APDLCollector
! Date: 2026-03-25

/PREP7
K,1,0,0,0
K,2,1,0,0
L,1,2
FINISH
```

## Integration Points

### 1. APDL Command Classes (apdl/_AllCommands.py)

The `Commands` class is the aggregation point for all APDL command groups. `APDLCollector` will wrap this class.

### 2. Command Execution Backend

The collector intercepts `run()` calls. The existing `Commands.run()` implementation remains for live execution scenarios (gRPC/gUI).

## Usage Example

```python
from apdl import Commands
from apdl._collector import APDLCollector

# Create collector wrapping existing Commands
collector = APDLCollector(Commands())

# Use context manager for processor scope
with collector.prep7():
    collector.k(1, 0, 0, 0)
    collector.k(2, 1, 0, 0)
    collector.l(1, 2)

# Write collected commands to file
collector.to_file("model.inp")
```

## Out of Scope

- Direct ANSYS execution (gRPC/gUI) — handled by existing infrastructure
- Command validation or syntax checking
- Partial script generation (one command at a time streaming to file)
